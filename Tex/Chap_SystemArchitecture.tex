\chapter{系统架构总体设计}\label{chap:architecture}

\section{总体设计目标}

针对嵌入式自主移动平台对 SLAM 系统在实时性、能耗以及精度方面的严苛要求，本文设计的基于 FPGA 的后端加速器旨在通过软硬协同的设计思路，突破传统通用处理器的计算瓶颈。系统的总体设计目标主要包含以下四个方面：

\begin{enumerate}
    \item \textbf{高实时性需求}：实时性是 SLAM 系统实现在线定位的关键指标。为了配合视觉前端的高频采样并确保控制回路的稳定性，本系统设计的后端加速器需实现在滑动窗口负载下，单次非线性优化的频率稳定在 \textbf{25~Hz 以上}。这意味着系统必须在 40~ms 内完成包括 Jacobi 矩阵生成、Hessian 矩阵累加及舒尔补消元在内的所有复杂矩阵运算。

    \item \textbf{低功耗与高能效比}：嵌入式平台（如无人机、手持 AR 设备）对续航时间极其敏感。本设计的目标是在维持高性能计算输出的同时，满足移动端的低功耗限制。通过利用 FPGA 的硬件定制特性，引入动态门控机制跳过无效计算，旨在实现显著优于通用嵌入式处理器（如 ARM Cortex-A 系列）的\textbf{能效比（Performance per Watt）}。

    \item \textbf{高定位精度维持}：硬件加速器的引入不应以牺牲定位精度为代价。由于 FPGA 硬件实现中通常涉及定点化（Fixed-point）处理，本系统的设计目标是优化位宽分配与运算溢出控制，确保经硬件加速后的后端优化结果（位姿轨迹与路标点坐标）与原始双精度浮点全软件实现相比，其误差保持在极小范围内，维持系统原有的\textbf{定位精度等级}。

    \item \textbf{资源利用率最优化}：本文采用的 Zynq-7000 或低端 UltraScale+ 系列 FPGA 片上资源（如逻辑单元 LUT、乘法器 DSP、块存储 BRAM）相对有限。因此，设计目标之一是实现计算逻辑的紧凑化。通过复用计算单元、优化 Hessian 矩阵的存储结构（如隐式索引）以及精简数据流水线，确保核心加速算子能够被适配到\textbf{资源受限的嵌入式 SoC} 中。
\end{enumerate}

\section{软硬件功能划分}

为了在 Zynq 异构 SoC 平台上实现最优的性能与能效平衡，本文遵循“计算密集型任务迁移至硬件，逻辑管理型任务保留在软件”的原则，对系统进行了深度的软硬件功能划分。系统整体分为处理器系统（Processing System, PS）端与可编程逻辑（Programmable Logic, PL）端两大部分。

\subsection{ARM 处理器端（PS）任务}

ARM 处理器端主要负责系统的全局协调、低频逻辑评估以及对外部传感器的管理。其具体任务包括：
\begin{itemize}
    \item \textbf{数据获取与多源同步}：通过驱动程序实时采集相机图像与 IMU 原始数据，并对异步传感器数据流进行精确的时间戳对齐。
    \item \textbf{视觉前端追踪}：执行特征点提取与基于光流法的帧间特征追踪，为后端优化提供关键帧及观测约束。
    \item \textbf{滑动窗口状态管理}：维护滑动窗口内关键帧的位姿（Pose）、速度（Velocity）以及传感器偏差（Bias）。
    \item \textbf{EKF 位姿更新}：基于 FPGA 端返回的压缩海森矩阵信息，执行增量式扩展卡尔曼滤波（EKF）位姿更新及系统协防差矩阵的维护。
\end{itemize}

\subsection{FPGA 逻辑端（PL）任务}

FPGA 逻辑端作为运算加速核心，利用其大规模并行计算能力处理算法中计算密度最高、数据流规整的矩阵运算单元。具体任务包括：
\begin{itemize}
    \item \textbf{Jacobi 矩阵并行生成}：针对每一条观测残差约束，利用 5 级深度流水线并行计算残差相对于位姿和路标点的雅可比偏导数。
    \item \textbf{Hessian 矩阵块累加}：实时将生成的雅可比矩阵进行外积运算并累加至对应的 Hessian 块区域，构建全局方程 $\mathbf{H}\Delta\mathbf{X} = \mathbf{b}$。
    \item \textbf{SOB 编码与动态门控逻辑}：读取稀疏观测位图（SOB），在硬件层级实现条件计算触发。
    \item \textbf{Schur 消元核设计}：实现高并发的 $3 \times 3$ 矩阵求逆运算以及对称 Hessian 矩阵更新模块，高效完成路标点的消去。
\end{itemize}

\section{数据流分析与通信协议设计}

\subsection{数据流分析}

SLAM 后端优化的数据流主要由视觉观测残差、当前状态初值以及 IMU 预积分约束组成。数据首先由 PS 端从传感器缓冲区提取并进行预处理，随后通过 AXI 总线分发至 PL 端的各个加速模块。

\subsection{PS-PL 通信协议}

为了确保高带宽、低延迟的数据交互，本系统设计了一套专用的通信协议：
\begin{enumerate}
    \item \textbf{寄存器控制流}：采用 AXI-Lite 接口，用于 PS 端对加速器进行初始化配置、启动信号下发以及运行状态的回读。
    \item \textbf{高速数据流}：采用 AXI-Stream 接口结合直接存储器访问（DMA）技术。
    \item \textbf{中断触发机制}：加速器完成计算后，通过 PL-to-PS 中断信号告知 ARM 处理器，触发中断处理程序读取计算结果。
\end{enumerate}
