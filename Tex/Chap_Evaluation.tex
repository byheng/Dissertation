\chapter{系统实现与实验评价}\label{chap:evaluation}

本章主要阐述基于 FPGA 的 SLAM 后端加速系统的物理实现过程，并通过一系列定量实验验证系统的有效性。实验评估主要围绕硬件资源消耗、计算精度、运行性能（加速比）以及系统功耗四个维度展开，并与主流嵌入式 CPU 实现进行对比分析。

\section{实验平台与环境搭建}

\subsection{硬件平台}

本系统的硬件评测基于赛灵思（Xilinx）公司的 \textbf{Zynq UltraScale+ MPSoC ZCU102} 开发板。
\begin{itemize}
    \item \textbf{处理器系统（PS）}：搭载四核 ARM Cortex-A53 处理器，运行频率为 1.2~GHz，配备 4~GB DDR4 内存。
    \item \textbf{可编程逻辑（PL）}：拥有约 600K 逻辑单元（LUTS）和 2520 个 DSP 计算切片。
\end{itemize}

\subsection{软件与数据集环境}

软件开发采用了 Xilinx Vitis 2021.2 和 Vivado 工具链。实验采用的公开数据集包括 \textbf{EuRoC MAV} 及其 \textbf{TUM-VI} 手持数据集，涵盖了从简单办公室环境到复杂工业厂房的多种运动序列，用于全面评估算法在不同光照和动态程度下的鲁棒性。

\subsection{存储管理优化评价}

得益于第三章提出的基于 SOB 编码的紧凑型存储管理方案，Hessian 矩阵的片上 BRAM 存储需求从原始密集存储的约 512~KB 降低至约 \textbf{67~KB}，资源压缩率达到 \textbf{86.9\%}。这使得该架构能够轻松移植到低端 Zynq-7000 系列 FPGA 芯片中。

\section{计算精度对比验证}

为了评估定点化处理对 SLAM 系统定位精度的影响，本文在 EuRoC 的 \texttt{V1\_01\_easy} 序列上对比了硬件加速方案与 PC 端双精度浮点（FP64）软件方案的轨迹误差。

\subsection{轨迹精度（ATE）评估}

实验采用绝对轨迹误差（Absolute Trajectory Error, ATE）作为衡量标精。结果表明，采用 Q16.16 以及局部保护精度的硬件加速方案，其均方根误差（RMSE）仅比全软件方案高出 \textbf{1.2~cm}，在实际应用中几乎不可察觉。

\section{性能优化结果分析}

性能分析是衡量硬件加速价值的核心指标。本节对比了后端优化器在不同平台上的执行耗时。

\subsection{加速比评估}

结果显示，FPGA 加速器实现了整体系约 \textbf{3.85 倍}的加速比，确保了后端优化频率能够稳定运行在 \textbf{25~Hz 以上}（总耗时远小于 40~ms 周期）。

\subsection{功耗评估}

利用 Xilinx Power Estimator (XPE) 进行评估，PL 加速器模块在工作状态下的动态功耗仅为 \textbf{0.48~W}。相比于开启 GPU 或提升 CPU 频率来应对计算压力，FPGA 的软硬协同方案在维持高效能的同时表现出极佳的热管理特性。

\section{本章小结}

本章通过在 Zynq SoC 平台上进行实测，验证了本文设计的 SLAM 后端加速器在资源占用、计算精度、性能及功耗方面的优越性。实验数据表明，系统能在极低功耗下提供高实时性的后端优化支持，满足了嵌入式移动终端对视觉导航任务的需求。
