## 4. IMU 噪声标定

### 4.1 准备 IMU 配置文件

**话题名称：** `/imu`

**配置文件：** `imu.yaml`

创建命令：
```bash
cat > ~/kalibr_ws/imu.yaml << 'EOF'
#Accelerometers
accelerometer_noise_density: 0.01   # [m/s^2/sqrt(Hz)] 初始估计值
accelerometer_random_walk: 0.0002   # [m/s^3/sqrt(Hz)] 初始估计值

#Gyroscopes
gyroscope_noise_density: 0.001      # [rad/s/sqrt(Hz)] 初始估计值
gyroscope_random_walk: 0.00001      # [rad/s^2/sqrt(Hz)] 初始估计值

rostopic: /imu                      # IMU 话题名
update_rate: 200.0                  # IMU 采样率 [Hz]
EOF
```

完整内容：
```yaml
#Accelerometers
accelerometer_noise_density: 0.01   # [m/s^2/sqrt(Hz)] 初始估计值
accelerometer_random_walk: 0.0002   # [m/s^3/sqrt(Hz)] 初始估计值

#Gyroscopes
gyroscope_noise_density: 0.001      # [rad/s/sqrt(Hz)] 初始估计值
gyroscope_random_walk: 0.00001      # [rad/s^2/sqrt(Hz)] 初始估计值

rostopic: /imu                      # IMU 话题名
update_rate: 200.0                  # IMU 采样率 [Hz]
```

**参数说明：**
- `accelerometer_noise_density`: 加速度计噪声密度（白噪声）
- `accelerometer_random_walk`: 加速度计偏置随机游走
- `gyroscope_noise_density`: 陀螺仪噪声密度（白噪声）
- `gyroscope_random_walk`: 陀螺仪偏置随机游走
- `update_rate`: IMU 采样频率
- 这些是**初始估计值**，联合标定后会得到精确值

### 4.2 IMU 噪声参数说明

**重要：** Kalibr **不支持**单独的 IMU 噪声标定！

IMU 噪声参数只能通过以下方式获取：

1. **从 IMU 数据表获取**（推荐） - 查看 IMU 芯片的数据手册
2. **通过联合标定估计** - `kalibr_calibrate_imu_camera` 会自动优化这些参数
3. **使用其他工具** - 如 Allan 方差分析等

**本次标定：**
- 我们在 `imu.yaml` 中填入了初始估计值
- 联合标定会自动优化这些参数并输出精确值
- **不需要**单独的 IMU 噪声标定步骤

**关于 imu_noise.bag：**
- 我们录制了这个文件，但 **Kalibr 无法单独使用它**
- 可以用其他工具（如 allan_variance_ros）分析
- 或者直接使用初始估计值进行联合标定

## 5. 相机-IMU 联合标定

### 5.1 录制联合数据

**录制命令：**
```bash
rosbag record /cam0/image_rect /cam1/image_rect /imu -O ~/kalibr_ws/stereo_imu.bag
```

**录制要求（非常重要！）：**

**时长：** 60-90 秒

**运动要求：**
1. **必须做丰富的 3D 运动：**
   - 绕 X、Y、Z 三个轴旋转（摇头、点头、侧翻）
   - 前后、左右、上下平移
   - **关键：必须有足够的线性加速度**（不能只转动，要有推拉）

2. **保持棋盘格可见：**
   - 棋盘格在两个相机视野内
   - 可以部分遮挡，但要能检测到角点

3. **运动平滑：**
   - 不要太快（避免模糊）
   - 不要太慢（要有足够激励）

4. **开始和结束：**
   - 开始前静止 2-3 秒
   - 结束前静止 2-3 秒

**推荐运动模式：**
- 手持设备对着棋盘格
- 做 "∞" 字形运动（横8字）
- 同时加入上下、前后移动
- 适当加入旋转倾斜

**数据验证：**
```bash
rosbag info ~/kalibr_ws/stereo_imu.bag
```

本次录制结果：
- 时长：2分28秒 (148秒)
- 文件大小：1.4 GB
- 双目图像：各 2361 帧
- IMU 消息：33,375 条

### 5.2 运行 IMU-相机联合标定

**命令：**
```bash
# 进入容器
docker exec -it SchurVINS_container bash

# 进入工作目录
cd ~/kalibr_ws

# 运行联合标定
~/kalibr_ws/devel/lib/kalibr/kalibr_calibrate_imu_camera \
    --bag stereo_imu.bag \
    --cam stereo_calib-camchain.yaml \
    --imu imu.yaml \
    --target checkerboard.yaml
```

**标定时间：** 约 10-20 分钟（比纯相机标定更耗时）

**标定过程：**
1. 提取所有图像的棋盘格角点
2. 优化相机-IMU 时间同步
3. 计算相机到 IMU 的 6D 外参（旋转 + 平移）
4. 精确估计 IMU 噪声参数

### 5.3 标定结果

**生成文件：**
- `stereo_imu-camchain-imucam.yaml` - 完整标定结果（相机+IMU）
- `stereo_imu-imu.yaml` - 精确的 IMU 噪声参数
- `stereo_imu-results-imucam.txt` - 详细标定报告
- `report-imucam-stereo_imu.pdf` - PDF 报告（可选）

## 6. 常见问题与解决方案

### 6.1 容器内缺少依赖

**问题：** `ModuleNotFoundError: No module named 'xxx'`

**解决：**
```bash
# igraph
apt-get install -y python3-igraph

# scipy
apt-get install -y python3-scipy

# wxPython (GUI 相关)
apt-get install -y python3-wxgtk4.0
```

### 6.2 话题名称错误

**症状：** 录制的 bag 文件很小（几 KB）

**解决：**
1. 先用 `rostopic list` 查看实际话题名称
2. 确认话题正在发布：`rostopic echo /话题名 -n 5`
3. 使用正确的话题名称录制

### 6.3 GUI 显示问题

**问题：** `Unable to access the X Display`

**解决：**
- 如果不需要 GUI：添加 `--dont-show-report` 参数
- 如果需要 GUI：配置 X11 转发（已配置）

### 6.4 标定板检测失败

**症状：** 标定过程中没有检测到角点

**可能原因：**
- 棋盘格尺寸配置错误（检查 `targetCols` 和 `targetRows`）
- 图像曝光过暗或过亮
- 运动模糊
- 棋盘格未完全在画面内

## 8. 生成 SchurVINS 配置文件

标定完成后，需要将 Kalibr 的输出转换为 SchurVINS 格式。

### 8.1 使用自动转换脚本

**转换脚本位置：** `/catkin_ws/src/SchurVINS/svo_ros/scripts/kalibr_to_svo.py`

**转换步骤：**

1. **准备 IMU 参数文件**（因为 Kalibr 输出的 IMU 文件格式需要调整）

```bash
# 在容器内创建转换用的 IMU 文件
docker exec SchurVINS_container bash -c 'cat > /root/kalibr_ws/imu_for_conversion.yaml << EOF
accelerometer_noise_density: 0.01
accelerometer_random_walk: 0.0002
gyroscope_noise_density: 0.001
gyroscope_random_walk: 1.0e-05
update_rate: 200.0
EOF'
```

2. **运行转换脚本**

```bash
# 进入容器
docker exec -it SchurVINS_container bash

# 进入标定数据目录
cd ~/kalibr_ws

# 运行转换脚本
python3 /catkin_ws/src/SchurVINS/svo_ros/scripts/kalibr_to_svo.py \
    --kalibr stereo_imu-camchain-imucam.yaml \
    --imu imu_for_conversion.yaml
```

这将生成 `svo_stereo_imu-camchain-imucam.yaml` 文件。

3. **复制生成的配置文件到本地**

```bash
docker cp SchurVINS_container:/root/kalibr_ws/svo_stereo_imu-camchain-imucam.yaml ./
```

### 8.5 运行 SchurVINS

SchurVINS 提供了两种模式的启动文件：

#### 8.5.1 前端模式（Frontend Only）

适用于快速测试和调试，不使用后端优化。

**启动文件：** `simit_stereo_imu.launch`

```bash
# 进入容器
docker exec -it SchurVINS_container bash

# 进入工作空间
cd /catkin_ws

# Source 环境
source devel/setup.bash

# 启动 SchurVINS 前端
roslaunch svo_ros simit_stereo_imu.launch
```

#### 8.5.2 完整 VIO 模式（Frontend + Backend）

包含 Ceres 后端优化，提供更高精度的位姿估计。

**启动文件：** `simit_vio_stereo.launch`
**位置：** `/catkin_ws/src/SchurVINS/svo_ros/launch/simit_vio_stereo.launch`

```xml
<!-- Launch File for running SchurVINS with ceres backend using simit stereo+IMU calibration -->
<launch>
  <arg name="calib_file" default="$(find svo_ros)/param/calib/simit_stereo.yaml"/>

  <!-- SVO node -->
  <node pkg="svo_ros" type="svo_node" name="svo" clear_params="true" output="screen" args="--v=0">

    <!-- Camera topics to subscribe to -->
    <param name="cam0_topic" value="/cam0/image_rect" type="str" />
    <param name="cam1_topic" value="/cam1/image_rect" type="str" />

    <!-- IMU topic to subscribe to -->
    <param name="imu_topic" value="/imu" type="str" />

    <!-- Camera calibration file -->
    <param name="calib_file" value="$(arg calib_file)" />

    <!-- Parameters -->
    <rosparam file="$(find svo_ros)/param/vio_stereo.yaml" />
    <param name='runlc' value='false' />

  </node>

  <!-- RVIZ -->
  <node name="vis" pkg="rviz" type="rviz" args=" -d $(find svo_ros)/rviz_config_vio.rviz" />

</launch>
```

**启动命令：**

```bash
# 进入容器
docker exec -it SchurVINS_container bash

# 进入工作空间
cd /catkin_ws

# Source 环境
source devel/setup.bash

# 启动完整 VIO 系统
roslaunch svo_ros simit_vio_stereo.launch
```

**配置说明：**
- `use_ceres_backend`: 在 `vio_stereo.yaml` 中设置为 `true` 启用后端优化
- `runlc`: 设置为 `false` 禁用回环检测（可根据需要启用）
- RVIZ 配置文件：`rviz_config_vio.rviz`

#### 8.5.3 使用 rosbag 测试

```bash
# 在另一个终端播放 rosbag
rosbag play your_data.bag

# 或指定播放速度
rosbag play -r 0.5 your_data.bag  # 0.5 倍速
```

**注意事项：**
- 确保话题名称与您的相机和 IMU 发布的话题一致
- 如果话题名称不同，修改 launch 文件中的对应参数
- 图像话题应该是已去畸变的 (rectified)
- IMU 频率应为 200 Hz（与标定时一致）
- 对于 VIO 模式，建议使用较高配置的计算机

### 9.8 可能需要重新标定的情况

如果出现以下情况，建议重新标定：

1. **重投影误差过大**（> 2 像素）
   - 原因：标定板检测不准确，图像模糊，标定板尺寸错误
   - 解决：重新录制数据，确保图像清晰，检查标定板参数

2. **IMU 归一化误差过大**（> 3.0）
   - 原因：运动激励不足，IMU 噪声参数不准确，时间同步问题
   - 解决：录制数据时做更丰富的 3D 运动，调整 IMU 噪声参数

3. **基线距离与实测差距过大**（> 5%）
   - 原因：标定板尺寸错误，相机同步问题
   - 解决：检查标定板实际尺寸，确保双目图像时间同步

4. **重力大小偏差过大**（偏离 9.81 m/s² 超过 5%）
   - 原因：加速度计校准不准确，坐标系错误
   - 解决：检查 IMU 传感器，重新标定
